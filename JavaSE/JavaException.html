    # JavaException

    # 什么是异常

    • 定义：程序执行过程中出现了不正常的情况，叫做异常；
    • java会将异常信息输出到控制台上。
      即jvm打印 Exception in thread "main" java.lang.ArithmeticException: / by zeroat com.notes.java se.exception.ExceptionTest01.main(ExceptionTest01.java:13)
    • 异常在Java中用类对象来表示。

    # 出现异常时会发生什么？

    • Java会调用异常构造方法new异常对象，并将异常信息输出到控制台。

    # 异常的分类和类继承结构图？

    • 分为编译时异常和运行时异常RuntimeException。
    • 编译时异常：因为编译时异常必须在编译(编写)阶段预先处理，若不处理编译器报错，因此得名。
    • 运行时异常？……
    • 二者区别：编译时异常发生概率较高，运行时异常发生概率较低，（1）对于高概率发生异常，一般在其运行前对其预编译处理，编译出错，则不能运行。（2）对于低概率发生异常，一般不预处理。
    • 共同点：都是在运行时出现的，因为调用异常对象要new对象。

    # 异常的两种处理方式？

    • 1.在方法声明的位置上，使用throws关键字，抛给上一级，谁调用我，就抛给谁，上抛后，上一级调用者也需要选择上抛或try..catch捕捉 若上抛到最后，到main方法都没有处理，到jvm，则会终止程序
    • 2.最终都是要使用try..catch语句进行异常的捕捉 [[throws关键字]] [[trycatch关键字]] throw手动抛出异常

    # 异常对象的使用

    • 1.获取异常对象简单的描述信息：
    String msg = exception.getMessage();
    • 打印异常追踪的堆栈信息
    exception.printStackTrace();

    # finally子句的使用

    • 用法，结合trycatch 使用不能单独使用。
    • 语法
    try{
	doSome();
}cathc(IOException e){
	e.printStackTrace();
}fianlly{
	//语句
}

    # finally使用的原因

    • 文件流的关闭，在try语句块中，若某句出现异常则会导致后面流关闭语句被跳过。
    • 所以将流关闭语句写在fianlly子句中，就会执行。
    • finally子句中的语句无论是否发生异常都会执行。
    • 常见于文件流的关闭。

    # 异常执行顺序

    • 对于try finally return 执行顺序

    • 先try后finally最后return执行，return执行后方法结束

    • 第一步：编写一个类继承Exception或者RuntimeException,

    • 第二步：提供两个构造方法，一个无参，一个有参String类型的

    • 重写之后的方法不能比重写之前的方法抛出更多(更宽泛)的异常，可以更少

    try{
	//正常代码语句区
	//发生了异常直接跳过区
}catch(异常名称 e){//e保存了创建的异常的对象内存地址
	//异常对象处理区
}
//以上执行后，这里继续执行

try{
	doSome();
//try语句块某句出现异常，则后续代码直接跳过
//若doSome方法发生了异常，则直接跳过此区
}catch(ClassNotFoundException e){
	e.printStackTrace();//打印堆栈信息
	
}

    # throws关键字的使用

    • 语法：在方法参数后面编写 一个/多个 异常
    public static void doSome () throws ClassNotFoundException{}
    • 表示该方法内包含需要抛出的某些编译时异常，上抛到调用该方法的父方法。
    • 父方法调用了上抛异常的方法后，也需要选择继续上抛还是捕捉。
    • 最终上抛到main方法后，就不能上抛了，因为上抛到JVM后会终止程序，所以在main方法中都需要对异常进行捕捉[[trycatch关键字]]，
    • 最终异常都是以捕捉输出异常信息来解决的
