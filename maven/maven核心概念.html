    # maven

    # maven核心概念

    1.pom.xml pom项目对象模型， 把一个项目当做一个模型使用，控制maven构建项目的过程，管理jar依赖

    2.约定目录结构 maven项目的目录和文件的位置都是规定的

    3.依赖管理 管理项目使用jar文件

    4.仓库管理 资源存放位置

    5.生命周期 构建项目的过程

    6.插件和目标 执行maven构建时，用的工具是插件

    # maven下载

    Maven – Welcome to Apache Maven (opens new window)

    # maven配置

    子目录 bin ： 执行程序， 主要是mvn.cmd

    conf: maven工具本身配置文件 setting.xml

    配置环境变量： 新建一个M2_HOME的名称，指定它的值是maven工具安装目录，bin之前的目录

    D:\Software3\apache-maven-3.8.8

    再把M2_HOME加入到path当中，再所有路径之间加入%M2_HOME%\bin

    注意需要配置JAVA_HOME， 指定JDK , JAVA_HOME=D:\Software\JDK

    控制台输入 mvn -v
出现以下
C:\Users\hkp>mvn -v
Apache Maven 3.8.8 (4c87b05d9aedce574290d1acc98575ed5eb6cd39)
Maven home: D:\Software3\apache-maven-3.8.8
Java version: 18.0.2.1, vendor: Oracle Corporation, runtime: D:\Software\JDK
Default locale: zh_CN, platform encoding: UTF-8
OS name: "windows 11", version: "10.0", arch: "amd64", family: "windows"
即配置正确

    # maven约定目录结构

    项目结构

    hello/
	--/src
	------/main		#放主程序Java代码和配置文件
	---------/java	#程序包和包中Java文件
	---------/resource	#java程序要使用的配置文件
	
	------/test		#放测试程序代码文件(可以没有)
	---------/java	#测试程序包和包中Java文件
	---------/resource	#测试java程序要使用的配置文件
	
	--/pom.xml		#maven核心文件

    # idea设置maven

    使用自己安装的maven，需要覆盖idea中默认的设置，让idea指定maven安装位置等信息。

    配置入口， 配置当前工程设置，file->settings->Build, Excution, Deployment->Build Tools -> Maven

    Maven-home-path maven安装目录

    User settings File maven安装目录conf/setting.xml配置文件

    local repository 本机仓库目录位置

    Build Tools-> Maven -- Runner

    VM Options : -DarchetypCatalog=internal

    JAE jdk位置
